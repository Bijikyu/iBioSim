1  Introduction

The example described in this tutorial constructs a simple model for the cI and cII genes and the PR and PRE promoters from the phage λ decsion circuit. This example illustrates many of the features of iBioSim.

2  SBML Editor

After starting iBioSim, complete the following steps to create an SBML model for this example:
  1. Select File → New → Project. Browse to desired path and create a project named demo.
  2. Select File → New → SBML Model. Enter lambda as the SBML model ID at which point an SBML editor will open.
  3. Highlight the default compartment, select Edit Compartment, and change its ID to Cytoplasm. Also, change the units to volume.
  4. Select Add Species and enter CI as the ID, The lambda repressor as the name, and change the units to mole. Select Add Species again and enter CI2 as the ID, CI dimer as the name, and change the units to mole.
  5. Select Add Parameter and enter nd as the ID, Number of molecules in dimer as the name, and change the units to dimensionless.
  6. Select Definitions/Types tab, and select Add Unit and enter per_second as the ID. Select Add to List, select second as the kind, change the exponent to −1, and click Add. Click Add in the Unit Definition Editor. Repeat these steps to create a per_second_mole unit (i.e., (second)−1(mole)−1).
  7. Select Main Elements tab. Select Add Reaction and enter Dimerize_CI as the ID, Reaction to dimerize CI as the name, and change reversible to true.
  8. Select Add Reactant and select CI as the species, change Stoichiometry to Stoichiometry math, and set its value to nd.
  9. Select Add Product and select CI2 as the species. Leave the stoichiometry as 1.
  10. Highlight kf and select Edit Selected Parameter, change kf to k2f, and change the units to per_second_mole. Highlight kr and select Edit Selected Parameter, change kr to k2r, and change the units to per_second.
  11. Select Use Mass Action, select Add, and select Save and Check SBML. There should be no errors.
  12. Highlight lambda.sbml, using right mouse button, select View Network. Highlight lambda.sbml, using right mouse button, select View in Browser.
  13. Go back to the SBML editor complete the construction of the chemical reaction network shown below:

3  GCM Editor

This section describes how to construct a GCM model for this example:
  1. Select File → New → Genetic Circuit Model. Enter CI_CII as the GCM model ID at which point a GCM editor will open.
  2. Select Add Promoter, enter PR as the ID, and press Ok. Next, add the PRE promoter in the same way.
  3. Select Add Species, enter CI as the ID, and press Ok. Next, add the CII species in the same way.
  4. Select Add Influence, change the input to CI, change the output to CII, change the promoter to PR, and the type to repression. Next, add an activation influence between CII and CI on promoter PRE.
  5. Select Save GCM, highlight CI_CII.gcm file, and right click to select View Genetic Circuit.

4  Analysis

The following instructions describe how to analyze the GCM file just created. The SBML file can also be simulated using the following steps.
  1. Select Save GCM, highlight CI_CII.gcm file, and right click to select Create Analysis View.
  2. In the newly opened window, select ODE. Also, in this window, change the time limit to 2100.0 and print interval to 10.0. Finally, select Save and Run at the bottom of the window.
  3. After the simulation completes, click on the graph tab.
  4. Double click on the graph to bring up the graph editor. Highlight Average, if not already highlighted, select CI and CII, change the Title to "ODE Simulation Results", change the X-Axis Label to "Time (seconds)", and change the Y-Axis Label to "Number of Molecules". Press the OK button. Click on Export and enter file name of ode.jpg.
  5. Select the simulation options tab again, select Monte Carlo, change the number of runs to 20, and set the simulation ID to ssa. Double click on the graph to bring up the graph editor. Open the ssa simulation directory, and highlight run-1. Select CI and CII, change Title to "SSA Simulation Results", change the X-Axis Label to "Time (seconds)", and change the Y-Axis Label to "Number of Molecules". Press the OK button. Click on Export and enter file name of ssa-1.jpg. Repeat these steps to generate graphs for the average ( average.jpg) and standard deviation (stddev.jpg). Note that you can use the "Deselect All" button to remove all items from the graph.
  6. Click on the parameter editor tab. Highlight the PR species, and select Edit Species. Select Custom for the initial amount of PR and change it to 5. Press the Save and Run button. Click on the graph tab and following the steps above, create the following plots average_10.jpg, run1_10.jpg, and stddev_10.jpg.
  7. Simulate your lambda model with BioSim using the ODE method rkf45 with a time limit of 2100 and print interval of 50. Make a note of the simulation time and plot CI2 and CII. Next, simulate using the Euler method. Make a note of the simulation time and add CI2 and CII from the euler results to your graph. How do the simulation times and results compare? Change the time step and rerun the Euler method. Repeat until the results match up well. What time step is required for a good match? How do the simulation times compare?

5  Abstraction

The example in this section illustrates abstraction-based synthesis.
  1. Press the Save SBML button.
  2. Create or open an analysis view on your lambda model.
  3. Select "None" and "Network", and press the Save and Run button. Count the number of species and reactions in your model.
  4. Select "Monte Carlo", set the Time Limit to 2100.0, Print Interval to 100.0, and Runs to 20. Press the Save and Run button and record the simulation time.
  5. Create a new analysis view for your lambda model and select "Abstraction" this time.
  6. In the abstraction tab, change rapid equilibrium conditions 1 and 2 to 1000.0 as well as QSSA condition to 1000.0. Select "Network" then save and run. Count the number of species and reactions in your model.
  7. Select "Monte Carlo", set the Time Limit to 2100.0, Print Interval to 100.0, and Runs to 20. Press the Save and Run button and record the simulation time.
  8. Select File → New → Graph and enter a name for your new top-level graph.
  9. Click on the graph and find the average simulation results from your orginal model and graph CII and CIt. Also, add to this graph from your abstracted model CII and CI. How well do they compare? Send me by email a jpg of your result.
  10. Go back to your analysis view in which you did abstraction and change all the conditions back to 0.1. Regenerate the network and record the number of species and reactions. Regenerate the simulation results and record the simulation time.
  11. Go back to the top-level graph which should have updated results. How does it compare now?

6  GCM Editor

This section gives an example using the GCM editor.
  1. Create a new project named "xor", select File → New → Genetic Circuit Model, and enter the name xor.
  2. Add species A and B of type constant. Be sure to fill in both the ID and Name fields.
  3. Add species Abar, Bbar, X, Y, and C of type normal. Again, fill in the ID and Name fields.
  4. Add an influence of type repression with input A and output Abar and dimer of 2. You have just created an inverter. Create another inverter from B to Bbar.
  5. Add a promoter named PX1 and another promoter named PX2.
  6. Add a repression influence from A to X and use the promoter PX1 with dimer of 2.
  7. Add a repression influence from Bbar to X and use the promoter PX2 with dimer of 2. You have just created a NAND gate.
  8. Create another NAND gate with inputs Abar and B and output Y.
  9. Finally, create another NAND gate with inputs X and Y and output C.
  10. Change the decay parameter to 0.01.
  11. Press the "Save GCM" and "Save as SBML" buttons.
  12. Create an analysis view for your xor.sbml file.
  13. Select the User Defined Data tab and click on the "Use User Defined Data" button.
  14. Add a data point for A at time step 1000 to go to 20.
  15. Add a data point for B at time step 2000 to go to 20.
  16. Add a data point for A at time step 3000 to go to 0.
  17. Add a data point for B at time step 4000 to go to 0.
  18. Select the options tab, select "abstraction", and set a time limit of 5000, a print interval of 200, and 20 runs.
  19. Select the abstraction tab, and add species C to the interesting species list.
  20. Press the "Save Parameters" button then "Save and Run" button.
  21. Create a graph that includes A, B, and C. Email a jpg of this graph to me.
  22. Close your analysis view and edit your xor.gcm. Try adjusting some of the global parameters. Remember to save both your gcm sbml files. Reopen your analysis view and re-run your simulation. Send me a few different graphs. Change the titles of the graphs and provide a description in your email of the graphs as to what you changed.
